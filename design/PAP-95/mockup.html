Revision 5 · board feedback

One conversation language, two surfaces

The conference room and task/issue comments should feel like the same product. Each surface already owns half of the right design, so the proposal cross-pollinates rather than rebuilds.

What changed in rev 5 (your feedback):
  • Plan cards now use a full state outline — violet while in review, green when approved, red when changes are requested. No left stripe — the whole border carries the colour. Same card appears in both the conference room and task comments.
  • Status chips carry their glyph icons — the PAP-75 ring/check marks ride inside each chip (in progress = half-filled blue ring, done = green check, etc.).
Blue is liveness

Human messages, sent chats, and in_progress tasks — all blue. Blue = "live right now."

Yellow is planning

A task's planning mode tints its composer yellow — "thinking, not acting yet." (Task only.)

Status is metadata

"Worked 2m", "updated this task", status changes — quiet rows on a rail, never bubbles.

Plans are documents

A plan is its own card with a state outline (violet → green → red), in either surface.
